# Data Provenance Registry

This directory defines provenance for all model inputs used by the Python analysis scripts.

## Structure
- `primary_sources.json`: Source registry of primary/official datasets.
- `raw/*.(html|csv|ods|xlsx|json|txt)`: Cached source artifacts fetched from source URLs.
- `source_artifacts.lock.json`: SHA-256 lockfile for fetched artifacts.
- `extracted_metrics.json`: Deterministically extracted metrics from source artifacts.
- `comparative_primary_metrics.json`: Reproducible cross-country metrics for comparative markdown reports.
- `model_inputs/*.json`: Input parameters consumed by each script, with embedded provenance metadata.
- `run_manifest.json`: Input/output lineage log for each script run.
- `ASSUMPTION_REGISTER.md`: Auto-generated register of every assumption/derived block in model inputs.
- `claims_catalog.json`: Curated list of published claims and their model-input paths.
- `CLAIM_TRACEABILITY.md`: Auto-generated claim-level traceability map.
- `report_source_policy.json`: Allowed-domain policy for comparative markdown research docs.
- `HALLUCINATION_AUDIT.md`: Audit of original hardcoded/hallucination-risk values and logic bugs.

## Provenance rules
- Every model input file must include `provenance` blocks on each data block.
- `kind` must be one of: `primary_source`, `official_statistic`, `administrative_record`, `derived`, `assumption`.
- `source_id` values must resolve to `primary_sources.json` for source-based blocks.
- `derived` blocks must list `source_ids` and method.
- `assumption` blocks must include:
  - `assumption_basis`
  - `assumption_class` in `{common_parlance, scenario_model, policy_anchor, proxy}`
  - `source_ids` (non-empty) pointing to reference material that grounds the assumption choice.

## Script behavior
Scripts load inputs from `research/data/provenance/model_inputs` and fail fast if provenance validation fails.
Model input files must also include a `generated` block (added by the builder script).

## Rebuild and verify workflow
1. Rebuild cached inputs from source fetch + extraction:
   - `python3 code/build_provenance_inputs.py --refresh-sources`
2. Verify cached inputs still match current extraction rules:
   - `python3 code/build_provenance_inputs.py --verify`
3. Audit extraction coverage (what is extracted vs assumptions/derived synthesis):
   - `python3 code/build_provenance_inputs.py --verify --audit`
4. Refresh source-hash baselines after manual review of upstream changes:
   - `python3 code/build_provenance_inputs.py --refresh-sources --bless-source-hashes`
5. Regenerate claim traceability map:
   - `python3 code/build_claim_traceability.py`
6. Review generated assumption register:
   - `research/data/provenance/ASSUMPTION_REGISTER.md`
7. Review generated claim traceability map:
   - `research/data/provenance/CLAIM_TRACEABILITY.md`
8. Audit comparative markdown sources against primary-domain policy:
   - `python3 code/audit_markdown_sources.py --check`
9. Rebuild comparative benchmark metrics from primary APIs:
   - `python3 code/build_comparative_primary_metrics.py`
8. Run analysis scripts:
   - `python3 code/student_loans.py`
   - `python3 code/rent_vs_own.py`
   - `python3 code/historical.py`
   - `python3 code/leasehold.py`
   - `python3 code/subscriptions.py`

## Hallucination guardrails
- Raw source artifacts are cached and content-hashed (`source_artifacts.lock.json`).
- Source hashes are pinned in `primary_sources.json` (`expected_sha256`) and build fails on hash drift unless explicitly blessed.
- Extracted values are generated by deterministic parsing rules (`extracted_metrics.json`), using dataset CSV endpoints where available.
- CSV endpoint extraction has HTML fallbacks for core required metrics when ONS dataset endpoints are temporarily rate-limited.
- Model inputs are regenerated from extraction output, not hand-edited at runtime.
- `--verify` fails if cached model inputs diverge from fresh extraction.
- Claim-level mappings from published stats to provenance blocks are generated in `CLAIM_TRACEABILITY.md`.
